# Rule Data — 온톨로지의 진짜 무게

"추론은 if + search다"라는 의심에서 시작해 **존재론적 엔지니어링**까지 닿는 4층 구조의 정직한 해체.

전남 무안 한옥마을 한 사례로 보는 — **임계치 → 트리거 → 룰 작동 → 상태 변화**의 200ms 워크스루.

## 라이브 보기

GitHub Pages 활성화 시: <https://jessylimitless.github.io/ruledata/>

## 구성

- **Stage 01–04**: "추론"의 정체 — 결국 if + search라는 인정의 여정
- **Stage 05**: 4층 구조 (Ontological Engineering · Composable Rule Graph · Shared Semantic Substrate · Data Sources)
- **Stage 06**: 한옥마을 추가 한 줄에 담긴 5가지 존재론적 결정 + 철학(Aristotle/Quine) ↔ CS(Gruber/Palantir) 연결
- **Stage 07**: 무안 한옥마을 #B-3021 시나리오 — 룰 데이터 설정 → 트리거 → 200ms 작동 흐름 → Before/After 상태판
- **Stage 08**: 최종 포지션 — *"온톨로지 시스템은 그 자체로 존재론적 엔지니어링의 산물"*

## 기술

단일 HTML · Tailwind (CDN) · 반응형 (모바일 최적화) · 외부 의존성 없음

브라우저에서 `index.html`을 열면 됩니다.
